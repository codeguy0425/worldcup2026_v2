Ranking Sort Order

Mathematical Qualification

qualified: true when ≤ 7 other teams can still reach ≥ my points — can't be pushed out of top 8.

const threats = entries.filter(o => { const maxPts = o.pts + (o.thirdLocked ? 0 : 3) return maxPts >= e.pts }).length e.qualified = threats < 8

Mathematical Elimination

eliminated: true when 8+ other teams are definitively ahead — their lowest possible (lose → stay at current pts) > my best possible.

const myMax = e.pts + (e.thirdLocked ? 0 : 3) const definitivelyAhead = entries.filter(o => { if (o === e) return false // Their worst = current pts (losing adds 0) return o.pts > myMax }).length e.eliminated = definitivelyAhead >= 8

thirdLocked Flag

Set when a team is confirmed as 3rd place in their group — either because the group is complete, or the team has an unassailable position.

thirdLocked: third.status != null || allPlayed

Note: != null catches both null and undefined (vs !== null which misses undefined).

Step 6–7 Bracket Computation & Resolution Placeholder Safety

Bracket JSON (Step 6)

// bracket.json — 6 rounds × 32 matches { "r32": [ { matchId: 73, team1Id: "2A", team2Id: "CAN", team1Resolved: false, team2Resolved: true }, ... ] }

Resolved teams show real names (e.g. CAN, SUI, ARG). Placeholders keep original format (e.g. 2A, 3E/F/G/I/J, W73).

Matches Resolution (Step 7)

After bracket computation, resolved team names are written back into matches.json so other pages (Schedule, MatchPage) show real names instead of placeholders.

// Build resolution map from bracket data const resolvedMap = {} for (const ms of Object.values(bracket.rounds)) { for (const bm of ms) { resolvedMap[bm.matchId] = { team1Id: bm.team1Id, team2Id: bm.team2Id } } } // Apply to cleanMatches for (const cm of cleanMatches) { const r = resolvedMap[cm.id] if (r && (r.team1Id !== cm.team1Id || r.team2Id !== cm.team2Id)) { cm.team1Id = r.team1Id cm.team2Id = r.team2Id } }

End-to-End Data Flow Timeline

🔄 Score Fetch

node scripts/fetch-scores.mjs

Pulls live scores from openfootball → writes updated scores to sample-data.json. Reports how many scores changed.

📐 Phase Detection

detectPhase(matches) → phase.json

Determines current tournament phase: group, r32, r16, qf, sf, third, final. Checks if all group matches are complete and if all groups are done.

📊 Group Standings

computeStandings(A–L) → groups/*.json

For each of 12 groups: compute stats, apply tiebreaker chain, run 3ⁿ scenario simulation (if incomplete), assign advanced / eliminated / confirmedFirst flags. Complete groups skip simulation.

⚽ Top Scorers + Third-Placed

computeTopScorers() + computeThirdPlaced()

Top scorers: aggregate goals from all played matches. Third-placed: rank 12 teams across groups, determine mathematical qualification, flag locked positions.

🏆 Bracket + Resolution

computeBracket() → bracket.json → resolve matches.json

Build 32-match knockout bracket (6 rounds). Resolve known team IDs (confirmed firsts, completed groups). Then propagate resolved names back into matches.json for Schedule/MatchPage display.
