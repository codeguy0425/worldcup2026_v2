architecture proposal

WC2026 Redesign

compute-on-update · serve-only · phase-aware
The update script does all the math once, outputs pre-computed JSON files,
and the React app just renders them.
No runtime computation in the browser. At all.
current vs proposed
Before & After

← Before

  • Raw matches.ts → browser computes 3n scenarios every render
  • All logic paths loaded always
  • Bracket placeholders like "1E/3A/B/C/D" resolved at runtime
  • computeStandings() re-runs on every React state change
  • Hard to know what phase is active

→ After

  • Script pre-computes everything → writes JSON files once
  • phase.json controls what exists — dead phases have no files
  • Bracket has real team names from the start
  • React = pure presentation: fetch JSON → render
  • Zero computation cost at runtime
