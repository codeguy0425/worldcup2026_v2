# Spike What It Validates Observable Output Status
001 Phase Detection Given raw match data (scores or null), determine current phase — group ongoing, group complete, R32, knockout, ended phase: "r32" + rules that triggered it NEXT
002 Standings + A/E 3ⁿ scenario simulation — standings sorted correctly, teams marked (A) or (E) or still in contention Group A table with pts, H2H, (A)/(E) badges PENDING
003 Third-Placed Top 8 Collect 3rd from all 12 groups, sort by rules, pick top 8 + ties Sorted table: rank · group · team · pts · GD · GF PENDING
004 Bracket Resolution Map placeholders ("1E", "3A/B/C/D") to real team names after group stage completes R32 bracket: actual team names, no placeholders PENDING
005 JSON Output Write all computed data as JSON files in the public/data/ structure ls public/data/ with valid JSON files PENDING

tooling
Approach
language
Node.js with minimal deps (or plain Python). Each script reads sample-data.json and prints results to stdout.
output
Every spike is a CLI you can run: node 002-standings/spike.mjs → readable tables printed to terminal
no react
Zero bundlers, zero JSX, zero Vite. Pure logic validation. React only comes in after all 5 spikes are validated.
disposable
Spikes get thrown away once they've paid their debt. The real app reimplements from scratch using the learnings.
